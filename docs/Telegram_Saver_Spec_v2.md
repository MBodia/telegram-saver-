# Telegram Saver — Специфікація v2

> Оновлена специфікація навчального SaaS-проєкту для Claude Code.
> Версія v2 — з підтвердженими рішеннями автора (Богдан, 2026-04-29).
> Попередня версія: `Telegram_Saver_Spec.pdf`.

---

## 1. Опис проєкту

**Telegram Saver** — це веб-додаток у поєднанні з Telegram-ботом, який допомагає користувачам зберігати та аналізувати **будь-який контент з Telegram** (посилання, скріншоти, PDF, текст з контекстом) у структуроване сховище з AI-резюме та пошуком.

### Проблема яку вирішує

Користувачі Telegram постійно зберігають корисний контент у "Saved Messages" або пересилають собі: статті, скріншоти, фото з текстом, PDF, цікаві повідомлення з каналів. Через деякий час там накопичуються сотні елементів без структури. Знайти потрібну інформацію стає неможливо.

### Як працює рішення

- Користувач **логіниться на сайті через Telegram Login Widget** (без email/пароля)
- Підключає бота, який починає працювати у приватному чаті
- Кидає боту **будь-що**:
  - Посилання → бот витягує контент сторінки, генерує резюме
  - Скріншот / фото → бот через AI читає текст з зображення (OCR + vision), описує що на ньому, генерує резюме
  - PDF → витягує текст, генерує резюме
  - Текстове повідомлення (з або без forward з каналу/чату) → AI аналізує контекст: про що йдеться, ключова думка, теги
  - Forward з каналу → зберігає посилання на оригінал + текст + аналіз
- На сайті бачить організований список з пошуком, фільтрами, тегами

### Цільова аудиторія

Активні користувачі Telegram, які регулярно зберігають різноманітний контент. Особливо: студенти, дослідники, маркетологи, IT-фахівці, консультанти. Окрема увага — українсько- та російськомовні користувачі.

> **Важливо:** проєкт будується насамперед як навчальний. Мета — опанувати повний стек SaaS-розробки. Питання комерціалізації — після завершення MVP.

## 2. Функціональність MVP

### Авторизація та акаунт

- **Вхід виключно через Telegram Login Widget** (без email/пароля)
- При першому вході автоматично створюється профіль на основі даних від Telegram (id, username, first_name, photo_url)
- Сторінка профілю з основною інформацією

### Telegram-бот

Бот працює у приватному чаті. Користувач може:
- кидати контент напряму
- пересилати (forward) з будь-якого чату, каналу, або з "Saved Messages"

**Команди:**
- `/start` — вітання та інструкція
- `/help` — список можливостей
- `/list` — останні 5 збережень

**Бот приймає та обробляє:**
| Тип повідомлення | Що робить |
|---|---|
| Текст з URL | Витягує URL, парсить сторінку, аналізує текст повідомлення як контекст |
| Тільки текст | AI аналізує: про що, ключова думка, пропонує теги |
| Фото / скріншот | AI vision: OCR (читає текст), описує що на зображенні |
| PDF (до 20 МБ) | Витягує текст, генерує резюме |
| Forward з каналу/чату | Зберігає посилання на оригінал + автор + повний текст + аналіз |
| Альбом фото | Обробляє кожне фото окремо, групує в одне збереження |

Усі повідомлення супроводжуються підтвердженням ("Збережено: [заголовок]") або зрозумілою помилкою.

### Обробка посилань

- Заходить на URL і витягує метадані (заголовок, опис)
- Витягує основний текст статті (через `@mozilla/readability`)
- Через **Claude API** генерує коротке резюме (3-5 речень)
- Зберігає: URL, заголовок, опис, повний текст, резюме, дату, користувача

### Обробка зображень (нове в v2)

- Завантажує файл у Supabase Storage
- Через **Claude Vision API** (модель з підтримкою зображень) виконує:
  - OCR — витягує весь текст з картинки
  - Опис — що зображено (для пошуку)
  - Контекст — якщо в caption є коментар, інтегрує його
- Зберігає: шлях у Storage, витягнутий текст, опис, резюме

### Обробка PDF

- Завантажує файл у Supabase Storage
- Витягує текст через `pdf-parse`
- Через Claude API генерує резюме
- Зберігає метадані файлу та резюме в базу

### Аналіз тексту (нове в v2)

Якщо повідомлення — просто текст або forward без посилання:
- Claude API аналізує: тема, ключова думка, пропоновані теги (3-5)
- Зберігає оригінал тексту + аналіз

### Веб-інтерфейс

- **Головна сторінка:** список усіх збережень користувача
- **Кожен елемент:** заголовок, резюме/опис, дата, тип (link / photo / pdf / text / forward), мініатюра (для фото)
- **Пошук:** по заголовку, резюме, витягнутому тексту з зображень
- **Теги:** додавання вручну + AI-пропоновані з можливістю прийняти
- **Фільтри:** за тегами, типом контенту, датою
- **Сторінка деталей `/item/[id]`:** повний контент, оригінал (фото/посилання/PDF)
- **Видалення збережень**

### Що НЕ ВКЛЮЧЕНО у MVP (на потім)

- Мобільний додаток
- Спільні папки / командна робота
- Експорт у Notion, Obsidian
- Темна тема
- Платні плани (Stripe — після MVP)
- Автоматичне розпізнавання дублікатів
- Підтримка інших месенджерів
- Відео та аудіо повідомлення (тільки фото в MVP)

## 3. Технічний стек

| Компонент | Технологія | Призначення |
|---|---|---|
| Фронтенд + бекенд сайту | Next.js 14+ (App Router) | Сайт + API routes |
| Мова | TypeScript | Типізація |
| Стилі | TailwindCSS | Стилювання |
| База даних | Supabase (PostgreSQL) | Користувачі, збереження, теги |
| Авторизація | **Telegram Login Widget + Supabase custom JWT** | Вхід через Telegram |
| Сховище файлів | Supabase Storage | PDF + зображення |
| Telegram-бот | grammY (TypeScript) | Telegram Bot API |
| AI текст + резюме | Anthropic Claude API | Аналіз і резюме |
| AI зображення | Claude API (vision модель) | OCR + опис фото |
| Парсинг URL | `@mozilla/readability` + `jsdom` | Текст статей |
| Парсинг PDF | `pdf-parse` | Текст з PDF |
| Хостинг сайту | Vercel | Деплой Next.js |
| Хостинг бота | Railway | 24/7 процес |
| Контроль версій | Git + GitHub | Історія коду |

**Бюджет на старті:** $0–15/міс. Claude vision дорожчий за text-only — закладаємо +$2-5/міс при активному використанні.

## 4. Архітектура системи

### Сервіс 1: Веб-сайт (Next.js на Vercel)
- Лендінг з кнопкою "Login with Telegram"
- Особистий кабінет зі списком збережень
- Сторінка налаштувань
- API routes (включно з callback від Telegram Login Widget для верифікації підпису)

### Сервіс 2: Telegram-бот (Node.js на Railway)
- Long Polling до Telegram
- Команди `/start`, `/help`, `/list`
- Прийом текстів, фото, PDF, forward
- Виклик Claude API (text + vision)
- Запис у спільну Supabase базу

### Спільна база: Supabase
Обидва сервіси читають/пишуть у ту саму базу.

### Як працює Telegram Login Widget (важливо!)

1. Користувач натискає кнопку "Login with Telegram" на сайті
2. Telegram відкриває віджет, користувач підтверджує
3. Telegram повертає на сайт дані: `id`, `first_name`, `username`, `photo_url`, `auth_date`, `hash`
4. **Сайт верифікує підпис `hash`** використовуючи токен бота (інакше можна підробити!)
5. Якщо підпис валідний — створюємо/знаходимо користувача в Supabase, видаємо custom JWT
6. Користувач залогінений

> ⚠️ Це складніше за email/пароль. Буде окремий етап (2a) присвячений тільки цьому.

## 5. Структура бази даних

База даних — 5 таблиць (одна нова для аналізу зображень).

### Таблиця: `profiles`
Розширена інформація про користувача.

**Поля:** `id` (uuid), `telegram_user_id` (bigint, унікальний — основний ідентифікатор), `telegram_username` (text), `telegram_first_name` (text), `telegram_photo_url` (text), `telegram_link_code` (text, для прив'язки бота, якщо вхід через Telegram Login робиться раніше), `created_at`, `updated_at`.

> Зміна v2: `telegram_user_id` тепер заповнюється одразу при першому вході (через Login Widget), а не пізніше.

### Таблиця: `saved_items`
Основна таблиця збережень.

**Поля:**
- `id` (uuid)
- `user_id` (uuid, FK → profiles)
- `type` (enum: `"link"`, `"pdf"`, `"photo"`, `"text"`, `"forward"`)
- `source_url` (text, nullable) — для посилань і forward
- `storage_path` (text, nullable) — для PDF та фото
- `title` (text)
- `description` (text)
- `full_text` (text) — текст статті / витягнутий з фото / повний текст повідомлення
- `summary` (text) — AI-резюме
- `ai_analysis` (jsonb, nullable) — структура: `{topic, key_idea, suggested_tags}` для текстових/forward
- `forward_origin` (jsonb, nullable) — інфо про канал/чат звідки переслано
- `created_at`

### Таблиця: `tags`
**Поля:** `id`, `user_id`, `name`, `color`, `created_at`.

### Таблиця: `item_tags`
Many-to-many. **Поля:** `item_id`, `tag_id`. Composite PK.

### Таблиця: `media_files` (нове в v2)
Для альбомів фото та множинних файлів в одному збереженні.

**Поля:** `id` (uuid), `item_id` (FK → saved_items), `storage_path` (text), `mime_type` (text), `extracted_text` (text — OCR), `vision_description` (text), `order_index` (int), `created_at`.

> Чому окрема таблиця: альбом може містити 5 фото — кожне зі своїм OCR-текстом, але групуються в одне збереження.

### Row Level Security (RLS)
Обов'язково для всіх таблиць. Користувач бачить тільки свої записи.

## 6. Поетапний план реалізації

> Зміна v2: 7 етапів замість 6. Етап 2a виділений під Telegram Login Widget.

### Етап 1: Базовий Next.js сайт
- Ініціалізувати Next.js 14+ з TypeScript і TailwindCSS
- Просту головну сторінку з заголовком "Telegram Saver"
- Git-репозиторій на GitHub
- Деплой на Vercel
- **Результат:** робочий сайт у інтернеті.

### Етап 2: База даних (Supabase)
- Створити проєкт на Supabase
- Створити таблиці: `profiles`, `saved_items`, `tags`, `item_tags`, `media_files`
- Налаштувати RLS-політики
- Підключити Supabase до Next.js (через `@supabase/ssr`)
- **Результат:** база готова, з'єднання працює.

### Етап 2a: Авторизація через Telegram Login Widget ⭐ НОВЕ
- Створити бота через @BotFather (потрібен токен для верифікації підпису)
- Додати Telegram Login Widget на головну сторінку
- API route `/api/auth/telegram` — верифікація підпису від Telegram
- Створити custom JWT для Supabase
- Створити trigger для авто-створення `profiles` при першому вході
- Сторінки: `/login` (з віджетом), `/dashboard` (приватна)
- **Результат:** можна залогінитись через Telegram, побачити свій username на дашборді.

### Етап 3: Telegram-бот (мінімум)
- Окремий Node.js проєкт з grammY
- Команда `/start` — вітання
- Команда `/help`
- Прив'язка до акаунту (по telegram_user_id, який вже є з Login Widget — не потрібен код прив'язки!)
- Деплой на Railway
- **Результат:** бот живе у Telegram, бачить хто йому пише.

> Зміна v2: оскільки telegram_user_id ми отримуємо одразу при логіні через Widget, прив'язка по коду не потрібна — бот просто шукає користувача за telegram_user_id з повідомлення.

### Етап 4: Збереження посилань і тексту
- Бот розпізнає URL у тексті
- Завантажує і парсить через `@mozilla/readability`
- Якщо немає URL — зберігає як `type="text"`
- Якщо forward — зберігає `forward_origin` і `type="forward"`
- Зберігає в `saved_items` (поки без AI)
- На сайті: список збережень
- **Результат:** кидаєш боту посилання чи текст — бачиш на сайті.

### Етап 5: Claude AI — резюме та аналіз тексту
- Підключити Anthropic API
- Для посилань → резюме статті
- Для тексту/forward → аналіз: тема, ключова думка, пропоновані теги
- Покращити UI: картки з резюме, дата, тип
- Сторінка деталей `/item/[id]`
- Видалення
- **Результат:** повноцінний продукт для текстового/посилального контенту з AI.

### Етап 6: Обробка зображень ⭐ НОВЕ
- Бот приймає фото (одиночні + альбоми)
- Завантаження в Supabase Storage
- Claude Vision API:
  - OCR → текст з картинки
  - Опис → що на зображенні
- Зберігання в `saved_items` (`type="photo"`) + `media_files`
- На сайті: мініатюри, перегляд, пошук по витягнутому тексту
- **Результат:** скріншоти і фото повноцінно обробляються.

### Етап 7: PDF, пошук, теги — фіналізація MVP
- PDF (до 20 МБ) → Storage → `pdf-parse` → резюме
- Пошук на сайті (по `title`, `summary`, `full_text`)
- Створення тегів вручну
- AI-пропоновані теги — кнопка "Прийняти"
- Фільтр за тегами і типом
- Тестування всіх сценаріїв
- **Результат:** завершений MVP v2.
