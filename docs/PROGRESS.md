# Telegram Saver — Прогрес і подальший напрямок

> Останнє оновлення: 2026-05-03 (Етап 7 завершено)

---

## Що вже зроблено

### Етап 1 — Базовий сайт ✅
- Next.js 16 з TypeScript і TailwindCSS
- Лендінг із описом продукту
- Деплой на Vercel

### Етап 2 — База даних ✅
- Supabase проєкт підключено
- 5 таблиць: `profiles`, `saved_items`, `tags`, `item_tags`, `media_files`
- RLS-політики для всіх таблиць
- Схема: `docs/database.sql`

### Етап 2a — Авторизація через Telegram ✅
- Telegram Login Widget на сайті
- Верифікація підпису hash від Telegram (`/api/auth/telegram`)
- Авторизація через бота — команда `/login` генерує 6-значний код, сайт перевіряє (`/api/auth/login`)
- Cookie-сесія (`ts_session`, 30 днів)
- Захищені сторінки: `/dashboard`, `/item/[id]`

### Етап 3 — Telegram-бот (мінімум) ✅
- Окремий проєкт `telegram-saver-bot` на grammY
- Команди: `/start`, `/help`, `/list`
- Деплой на Railway

### Етап 4 — Збереження посилань і тексту ✅
- Бот розпізнає URL, парсить через `@mozilla/readability`
- Тип `text` — звичайні повідомлення
- Тип `forward` — пересилки з каналів, зберігає `forward_origin`
- Список збережень на дашборді з пошуком і фільтрами за типом/тегом

### Етап 5 — Claude AI ✅
- Anthropic Claude Haiku — резюме і аналіз
- Посилання → резюме статті
- Текст/forward → тема, ключова думка, пропоновані теги
- Сторінка деталей `/item/[id]`
- Видалення збережень

### Дашборд — додаткові UX-функції ✅ (2026-05-03)
- **★ Обрані** — зірочка на кожній картці, жовтіє миттєво
- **🗑 Видалення** прямо з дашборду (без переходу на деталь)
- **Режим вибору** — чекбокси + панель масових дій (видалити / додати в обрані)
- Фільтр **"★ Обрані"** у тулбарі
- Колонка `is_favorite` в таблиці `saved_items`

### Етап 6 — Обробка зображень ✅ (повна реалізація, 2026-05-04)
- **Одиночне фото**: Claude Vision OCR + опис, Storage upload, `storage_path` + `source_url` в `saved_items`, запис в `media_files`
- **Альбоми**: буфер `albumBuffer` + таймер 1.5 сек збирає всі фото, кожне аналізується окремо, усі зберігаються в `media_files`, одне `saved_items` на альбом
- **Фото-документ** (без стиснення): аналогічно одиночному фото
- **Сайт — мініатюри**: `source_url` завантажується на дашборді, показується `<img>` 64×64 для photo-карток
- **Сайт — галерея**: `/item/[id]` показує сітку для альбому (з `media_files`), одне фото для одиночного
- **Сайт — OCR альбому**: кожен extracted_text з `media_files` показується окремо
- **Сторінка login**: повний редизайн — темна тема, анімації, Syne+Playfair, українська мова

### Етап 7 — PDF, пошук, теги, дата ✅ (2026-05-03)
- **PDF**: бот парсить через `pdf-parse`, Claude робить резюме, сайт показує кнопку "Відкрити PDF"
- **Покращений пошук**: дашборд шукає по `title`, `summary`, `description` і **`full_text`** (текст PDF/фото)
- **Ручне створення тегів**: форма "Додати свій тег" на сторінці `/item/[id]`
- **AI-пропоновані теги**: кнопка "Прийняти" з пропозиціями Claude на сторінці `/item/[id]`
- **Фільтр за датою**: кнопки "Сьогодні / Тиждень / Місяць" на дашборді
