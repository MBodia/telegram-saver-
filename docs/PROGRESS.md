# Telegram Saver — Прогрес і подальший напрямок

> Останнє оновлення: 2026-05-03

---

## Що вже зроблено

### Етап 1 — Базовий сайт ✅
- Next.js 16 з TypeScript і TailwindCSS
- Лендінг із описом продукту
- Деплой на Vercel

### Етап 2 — База даних ✅
- Supabase проєкт підключено
- 5 таблиць: `profiles`, `saved_items`, `tags`, `item_tags`, `media_files`
- RLS-політики для всіх таблиць
- Схема: `docs/database.sql`

### Етап 2a — Авторизація через Telegram ✅
- Telegram Login Widget на сайті
- Верифікація підпису hash від Telegram (`/api/auth/telegram`)
- Авторизація через бота — команда `/login` генерує 6-значний код, сайт перевіряє (`/api/auth/login`)
- Cookie-сесія (`ts_session`, 30 днів)
- Захищені сторінки: `/dashboard`, `/item/[id]`

### Етап 3 — Telegram-бот (мінімум) ✅
- Окремий проєкт `telegram-saver-bot` на grammY
- Команди: `/start`, `/help`, `/list`
- Деплой на Railway

### Етап 4 — Збереження посилань і тексту ✅
- Бот розпізнає URL, парсить через `@mozilla/readability`
- Тип `text` — звичайні повідомлення
- Тип `forward` — пересилки з каналів, зберігає `forward_origin`
- Список збережень на дашборді з пошуком і фільтрами за типом/тегом

### Етап 5 — Claude AI ✅
- Anthropic Claude Haiku — резюме і аналіз
- Посилання → резюме статті
- Текст/forward → тема, ключова думка, пропоновані теги
- Сторінка деталей `/item/[id]`
- Видалення збережень

### Дашборд — додаткові UX-функції ✅ (2026-05-03)
- **★ Обрані** — зірочка на кожній картці, жовтіє миттєво
- **🗑 Видалення** прямо з дашборду (без переходу на деталь)
- **Режим вибору** — чекбокси + панель масових дій (видалити / додати в обрані)
- Фільтр **"★ Обрані"** у тулбарі
- Колонка `is_favorite` в таблиці `saved_items`

---

## Що залишилось

### Етап 6 — Обробка зображень 🔜
- Бот приймає фото (одиночні + альбоми)
- Завантаження в Supabase Storage
- Claude Vision API: OCR (текст з картинки) + опис зображення
- Збереження у `saved_items` (type=`photo`) + `media_files`
- На сайті: мініатюри, перегляд фото, пошук по OCR-тексту

### Етап 7 — PDF, пошук, теги — фіналізація MVP 🔜
- PDF (до 20 МБ) → Storage → `pdf-parse` → Claude резюме
- Покращений пошук (по `full_text` з зображень і PDF)
- Ручне створення тегів
- AI-пропоновані теги — кнопка "Прийняти"
- Фільтр за датою
- Тестування всіх сценаріїв
